# 1. Definição do MVP
O MVP (Minimum Viable Product) do sistema Saúde & Vida foca no fluxo crítico de vendas e no controle rigoroso de estoque, garantindo a operação básica e a saúde financeira do estabelecimento.

- **O que está dentro do MVP:** Processo completo de vendas (à vista e a prazo), consulta ágil de produtos, cadastro de clientes para fidelização, baixa automática de estoque após cada transação e emissão de relatórios básicos de movimentação.
- **O que está fora do MVP:** Gestão complexa de compras com fornecedores, lançamentos de contas a pagar de serviços fixos (luz, aluguel, internet) e transferências de mercadorias entre diferentes unidades/filiais.
- **Por que você fez essas escolhas:** O objetivo primordial é garantir que a farmácia consiga realizar vendas com segurança e não perca o controle do inventário físico (o que tem na prateleira), que é o maior ativo do negócio.

# 2. Regras de Negócio (mínimo: 5)
**RN01 —** Venda de medicamentos controlados exige obrigatoriamente a liberação do Farmacêutico responsável.  
**RN02 —** É estritamente proibida a venda de qualquer produto que apresente estoque zero no sistema.  
**RN03 —** A venda na modalidade "a prazo" é permitida apenas para clientes que não possuam dívidas ou títulos atrasados.  
**RN04 —** O sistema deve emitir um alerta visual automático quando o estoque de um item atingir seu nível mínimo configurado.  
**RN05 —** Toda concessão de desconto acima de 10% no valor total da venda exige a inserção da senha do Gerente.

# 3. Requisitos Funcionais (mínimo: 8)
**RF01 —** Pesquisar produtos através do nome ou leitura de código de barras.  
**RF02 —** Realizar o fechamento de vendas e emissão do respectivo comprovante/cupom.  
**RF03 —** Cadastrar novos clientes diretamente no balcão de atendimento.  
**RF04 —** Atualizar o saldo de estoque automaticamente após a conclusão de uma venda.  
**RF05 —** Registrar vendas a prazo automaticamente no módulo de Contas a Receber.  
**RF06 —** Validar os dados de receitas e CRM para a venda de medicamentos controlados.  
**RF07 —** Gerar relatório analítico dos produtos mais vendidos por período.  
**RF08 —** Realizar autenticação (login) com diferentes perfis de acesso (Atendente, Farmacêutico e Gerente).  

---

# 🛡 4. Requisitos Não Funcionais (mínimo: 4)
**RNF01 —** O sistema deve ser totalmente acessível via navegador Web (SaaS).  
**RNF02 —** O tempo de resposta nas consultas de produtos deve ser inferior a 2 segundos para garantir agilidade no atendimento.  
**RNF03 —** O sistema deve garantir a segurança e privacidade dos dados dos clientes, em total conformidade com a LGPD.  
**RNF04 —** A interface deve ser responsiva, adaptando-se perfeitamente a tablets e computadores de mesa.

# 6. Documentação dos Casos de Uso

## **UC01 — Efetuar Venda**
**Ator(es):** Atendente.  
**Descrição:** Registro de saída de produtos para o cliente final.  
**Pré-condições:** Usuário logado e produtos com estoque disponível.  
**Pós-condições:** Estoque atualizado e comprovante emitido.  
**Fluxo Principal:**
1. Atendente inicia a venda.  
2. Inclui os produtos desejados.  
3. Sistema valida o estoque de cada item.  
4. Processa o pagamento (**UC05**).  
5. Finaliza a venda e emite o cupom.  
**Relacionamentos:** Include UC05, UC09 | Extend UC03, UC04.

## **UC02 — Consultar Produto**
**Ator(es):** Atendente / Gerente.  
**Descrição:** Verificar informações de preço e quantidade.  
**Fluxo Principal:**
1. Usuário digita nome ou lê o código.  
2. Sistema exibe preço e saldo atual.  

## **UC03 — Validar Receita**
**Ator(es):** Farmacêutico.  
**Descrição:** Validação de CRM para medicamentos de uso controlado.  
**Relacionamentos:** Extend UC01.

## **UC04 — Cadastrar Cliente**
**Ator(es):** Atendente.  
**Descrição:** Registro de dados pessoais para histórico e vendas a prazo.  
**Relacionamentos:** Extend UC01.

## **UC05 — Processar Pagamento**
**Ator(es):** Atendente.  
**Descrição:** Recebimento do valor da venda.  
**Fluxo Principal:**
1. Escolhe a forma (Dinheiro/Cartão/Prazo).  
2. Confirma o recebimento.  
**Relacionamentos:** Include em UC01 | Extend UC06.

## **UC06 — Gerar Conta a Receber**
**Ator(es):** Sistema (Automático).  
**Descrição:** Criação de título financeiro para cobrança futura.  
**Relacionamentos:** Extend UC05.

## **UC07 — Repor Estoque**
**Ator(es):** Gerente.  
**Descrição:** Entrada de mercadorias no sistema.  
**Fluxo Principal:**
1. Seleciona o produto.  
2. Insere a quantidade conforme Nota Fiscal.  
3. Sistema atualiza o saldo total.  

## **UC08 — Emitir Relatório Diário**
**Ator(es):** Gerente.  
**Fluxo Principal:**
1. Define o período desejado.  
2. Sistema gera PDF com resumo de vendas e movimentação.  

## **UC09 — Realizar Login**
**Ator(es):** Todos os perfis.  
**Descrição:** Autenticação obrigatória para acesso às funcionalidades.  
**Relacionamentos:** Include em todos os Casos de Uso.  

## **UC10 — Ajustar Estoque Manual**
**Ator(es):** Gerente.  
**Descrição:** Correção pontual de estoque por motivo de quebra, furto ou vencimento.  
**Relacionamentos:** Include UC09.
