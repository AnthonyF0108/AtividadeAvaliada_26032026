1. Definição do MVP
O MVP foca no fluxo crítico de vendas e controle de estoque.

Dentro do MVP: Vendas (à vista e prazo), consulta de produtos, cadastro de clientes, baixa automática de estoque e relatórios básicos de vendas.

Fora do MVP: Gestão de compras com fornecedores, contas a pagar de serviços fixos (luz/aluguel) e transferências entre unidades.

Justificativa: Garantir que a farmácia consiga vender e não perca o controle do que tem em prateleira.

2. Regras de Negócio (RN)
RN01: Venda de controlados exige liberação do Farmacêutico.

RN02: Proibida venda de produto com estoque zero.

RN03: Venda a prazo permitida apenas para clientes sem dívidas atrasadas.

RN04: O sistema deve alertar quando o estoque atingir o nível mínimo.

RN05: Todo desconto acima de 10% exige senha do Gerente.

3. Requisitos Funcionais (RF)
RF01: Pesquisar produtos por nome ou código de barras.

RF02: Realizar vendas e emitir comprovante.

RF03: Cadastrar novos clientes no balcão.

RF04: Atualizar estoque automaticamente após venda.

RF05: Registrar vendas a prazo no Contas a Receber.

RF06: Validar receitas de medicamentos controlados.

RF07: Gerar relatório de produtos mais vendidos.

RF08: Realizar login com diferentes perfis de acesso.

4. Requisitos Não Funcionais (RNF)
RNF01: O sistema deve ser acessível via navegador Web.

RNF02: O tempo de resposta na busca de produtos deve ser inferior a 2s.

RNF03: O sistema deve garantir a segurança dos dados dos clientes (LGPD).

RNF04: Interface responsiva para tablets e computadores.

6. Documentação dos Casos de Uso (10 Casos)
UC01 — Efetuar Venda
Ator: Atendente.
Fluxo: 1. Inicia venda; 2. Inclui produtos; 3. Valida estoque; 4. Processa pagamento (UC05); 5. Finaliza e emite cupom.
Relacionamento: Include UC05, UC09.

UC02 — Consultar Produto
Ator: Atendente / Gerente.
Fluxo: 1. Digita nome/código; 2. Sistema exibe preço e saldo em estoque.

UC03 — Validar Receita
Ator: Farmacêutico.
Descrição: Validação de CRM para itens controlados.
Relacionamento: Extend UC01.

UC04 — Cadastrar Cliente
Ator: Atendente.
Descrição: Registro rápido de novos clientes.
Relacionamento: Extend UC01.

UC05 — Processar Pagamento
Ator: Atendente.
Fluxo: 1. Escolhe forma (Dinheiro/Cartão/Prazo); 2. Confirma recebimento.
Relacionamento: Include em UC01.

UC06 — Gerar Conta a Receber
Ator: Sistema.
Descrição: Cria título financeiro para vendas a prazo.
Relacionamento: Extend UC05.

UC07 — Repor Estoque
Ator: Gerente.
Fluxo: 1. Seleciona produto; 2. Insere quantidade da nota fiscal; 3. Atualiza saldo.

UC08 — Emitir Relatório Diário
Ator: Gerente.
Fluxo: 1. Define período; 2. Sistema gera PDF com resumo de vendas e estoque.

UC09 — Realizar Login
Ator: Todos.
Descrição: Autenticação obrigatória para usar o sistema.
Relacionamento: Include em todos os UCs.

UC10 — Ajustar Estoque Manual
Ator: Gerente.
Descrição: Correção de estoque por quebra ou vencimento.
Relacionamento: Include UC09.
